void decodeDirective ( chain  ch,
char *  tab 
)

this routine decode only seleral directives. All other directives are directly managed by lex.c In this routine, we manage this directives :

Parameters
chThe chain to analyse.
taba char array used to load code byte by byte.
Returns
nothing.
  • .word w1, ... wn : put n word in contiguous way.
  • .byte b1, ... bn : put n bytes in contiguous way.
  • .asciiz s1, ... sn : put n string in contiguous way.
  • .space n : put n bytes initalized to 0.

Routine to make fastly a instruction using string parameters.

Parameters
nameName of instruction, "ADD"
opBinary translation, "10000"
typeFrom enum (see header) R => 0, ...
operandDefine which operands are defined. For example, 0110 => Only rt and rd can be defined in assembly source code.
specialDefine automatically some operands. For example, sa=00000 in case of ADD.
Returns
A completed structure Inst, ready to be added to the instruction set chain.
